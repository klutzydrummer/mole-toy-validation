## Abstract

The paper introduces techniques enabling "dynamic chunking mechanism which automatically learns content- and context-dependent segmentation strategies learned jointly with the rest of the model." Their hierarchical network (H-Net) replaces traditional tokenization pipelines with a unified end-to-end approach operating at the byte level. Key findings include: byte-level H-Nets matching Transformer performance on BPE tokens; multi-stage hierarchies improving scaling with data; enhanced character-level robustness; and significant gains in languages with weaker tokenization (Chinese, code) and DNA sequences, showing "nearly 4x improvement in data efficiency over baselines."

---

## Architectural Overview

The hierarchical structure follows a pipeline across S stages. At each stage s:

### Equation (1) — Main Processing Flow

```
x̂ˢ = ℰˢ(xˢ),    ẑˢ = 𝓜(xˢ),    ẑˢ = 𝒟ˢ(zˢ)
```

Encoder processing, main network computation, and decoder reconstruction across hierarchical stages.

### Equation (2) — Chunking Operation

```
(xˢ⁺¹, pˢ) = chunk(x̂ˢ)
```

The chunking layer produces downsampled representations and boundary probabilities for subsequent processing.

### Equation (3) — Gated Residual / Dechunking with Residual Connection

```
zˢ = dechunk(ẑˢ⁺¹, pˢ) + linear(x̂ˢ)
```

The paper states: "we adopt the first approach – adding a projection (linear) only to the residual connection."

**CRITICAL DETAIL — Zero Initialization:** The paper explicitly states:
> "One additional detail is that this residual connection is initialized close to 0; earlier versions of H-Net found this to be an important detail, but it may be less important when combined with additional techniques such as LR modulation."

The `linear` here is a learned linear projection applied only to the residual path (not to the main dechunked output). It is initialized close to zero.

## Dynamic Chunking: Routing Module

### Equation (4) — Boundary Probability Computation (Learned Routing)

```
qₜ = W_q x̂ₜ
kₜ = W_k x̂ₜ
pₜ = (1/2)(1 − (qₜᵀ kₜ₋₁) / (‖qₜ‖ ‖kₜ₋₁‖)) ∈ [0, 1]
bₜ = 𝟙{pₜ ≥ 0.5}
```

Context: "The routing module implements this intuition through cosine similarity between adjacent encoder outputs. Given encoder outputs X̂, it calculates boundary probabilities pₜ and boundary indicators bₜ as follows."

**Sequence start:** "p₁ = 1.0 by definition, ensuring the sequence begins with a boundary."

**W_q / W_k Initialization (from official code `hnet/modules/dc.py`):**
```python
self.q_proj_layer = nn.Linear(d_model, d_model, bias=False)
self.k_proj_layer = nn.Linear(d_model, d_model, bias=False)
with torch.no_grad():
    self.q_proj_layer.weight.copy_(torch.eye(d_model))
    self.k_proj_layer.weight.copy_(torch.eye(d_model))
self.q_proj_layer.weight._no_reinit = True
self.k_proj_layer.weight._no_reinit = True
```

Both W_q and W_k are initialized to the identity matrix. The `_no_reinit = True` flag prevents any downstream re-initialization from overwriting this.

**Implementation note (from code):** The cosine similarity is computed as:
```python
cos_sim = torch.einsum(
    "b l d, b l d -> b l",
    F.normalize(self.q_proj_layer(hidden_states[:, :-1]), dim=-1),
    F.normalize(self.k_proj_layer(hidden_states[:, 1:]), dim=-1),
)
boundary_prob = torch.clamp(((1 - cos_sim) / 2), min=0.0, max=1.0)
# Force boundary probability of the first element to 1.0
boundary_prob = F.pad(boundary_prob, (1, 0), "constant", 1.0)
```

Note the index shift: `q` from token `t-1` and `k` from token `t` (i.e., adjacent comparison with q computed on the *earlier* token, k on the *later* token).

## Dynamic Chunking: Dechunking Module

### Equation (5) — EMA Smoothing

```
z̄ₜ = Pₜ ẑₜ + (1 − Pₜ) z̄ₜ₋₁
```

Context: "The smoothing module applies an exponential moving average (EMA) with the following definition." This creates "smooth interpolations between chunks" by blending discrete chunk representations with previous context based on boundary confidence scores Pₜ.

**Implementation (from `dc.py`):** The EMA is implemented by reusing the Mamba-2 scan kernel:
```python
dt = torch.log(1 / (1 - p)).to(self.dtype)
x = (hidden_states / dt[..., None]).to(self.dtype)
A = -torch.ones((self.nheads,), device=hidden_states.device, dtype=torch.float32)
b = p.to(self.dtype)
c = torch.ones_like(b)
out = mamba_chunk_scan_combined(...)
```

The step-mode EMA:
```python
result = p * current_hidden_states + (1 - p) * inference_params.last_value
```

### Equation (6) — Confidence Scoring

```
cₜ = pₜ^(bₜ) (1 − pₜ)^(1−bₜ) = { pₜ      if bₜ = 1
                                   { 1 − pₜ  if bₜ = 0
```

Context: "The coefficient c quantifies the routing module's confidence in its boundary decisions."

### Equation (7) — Straight-Through Estimator (STE)

```
ste(cₜ) = cₜ + stopgradient(1 − cₜ)
```

Context: "The Straight-Through Estimator (STE)...rounds confidence scores to 1.0 in the forward pass while maintaining continuous gradients during backpropagation." This stabilizes discrete choice learning.

### Equation (8) — Causal Expansion / Plug-Back

```
z̃ₜ = z̄_{∑ₖ₌₁ᵗ bₖ}
```

Context: "The upsampling operation repeats each compressed vector until the next boundary position, ensuring that each reconstructed position receives information from its most recent chunk."

**Implementation (from `dc.py`):**
```python
plug_back_idx = torch.cumsum(boundary_mask, dim=1) - 1  # (B, L)
out = torch.gather(
    out,
    dim=1,
    index=plug_back_idx.unsqueeze(-1).expand(-1, -1, self.d_model),
)
```

### Equation (9) — Upsampler Output

```
upsampler(z̄, c)ₜ = ste(cₜ) · z̃ₜ
```

Context: "Multiplying upsampled vectors by their confidence scores incentivizes the routing module to make confident, accurate decisions."

## Optimization: Ratio Loss

### Equation (10) — Targeted Compression Loss

```
ℒ_ratio = (N / (N−1)) · ((N−1) · F · G + (1−F) · (1−G))

F = (1/L) ∑ₜ₌₁ᴸ bₜ       (fraction of tokens selected as boundaries)
G = (1/L) ∑ₜ₌₁ᴸ pₜ       (average boundary probability)
```

Context: "we introduce a ratio loss to guide compression...where F represents the fraction of vectors actually selected, G denotes the average boundary probability, and N controls the target compression ratio." The loss achieves minimum value 1.0 when F = G = 1/N.

### Combined Training Objective

```
ℒ = ℒ_AR + α ∑ₛ₌₀^{S−1} ℒ_ratio^s
```

where α = 0.03 across all experiments.

## Encoder/Decoder Architecture

- "we always stick to four layers (or the equivalent of four Mamba layers) in each encoder/decoder network"
- "Mamba layers in ℰ or 𝒟 are without an MLP"
- Width must be monotonically non-decreasing through the hierarchy: D⁰ ≤ D¹ ≤ ... ≤ Dˢ
- To expand dimensions (Dˢ → Dˢ⁺¹): "we append all vectors with a shared trainable vector of dimension Dˢ⁺¹ − Dˢ"
- To reduce dimensions (Dˢ⁺¹ → Dˢ): "we take the first Dˢ dimensions from each vector"

---

## Normalization and Signal Propagation

- "placing an RMSNorm...at the end of each network component (ℰˢ, 𝒟ˢ, and 𝓜)" to balance residual stream magnitudes across hierarchical stages.
- Outer stages receive "proportionally higher learning rates than inner stages."

---

## Gradient Flow Through the Router

- The only explicit gradient-blocking mechanism is the STE in Equation (7): `ste(cₜ) = cₜ + stopgradient(1 − cₜ)`.
- There is no mention of gradient isolation or detaching gradients through the boundary probability computation itself. Gradients flow through pₜ into W_q and W_k.
- The ratio loss ℒ_ratio provides a differentiable signal via G (the average of pₜ) to guide the compression ratio toward the target 1/N.

---

## Autoregressive Inference Procedure

Token generation loop (from paper):
1. Pass the token through the encoder network.
2. Step the routing module to determine whether the token needs processing.
3. Step the main network **if necessary** (only on boundary tokens), and also step the dechunking layer.
4. Use the result of the dechunking layer to step the decoder network.

Routing module inference state:
- `has_seen_tokens`: (batch_size,) bool tensor — whether batch element has processed any tokens.
- `last_hidden_state`: (batch_size, d_model) — last hidden state (used for adjacent key comparison).

DeChunk inference state:
- `last_value`: (batch_size, d_model) — last EMA value.

---

## Key Design Quotes

- "maintaining intact gradient flow" through the main computational path by applying linear projections only to residual connections, not main network inputs.
- "this residual connection is initialized close to 0; earlier versions of H-Net found this to be an important detail"
- "p₁ = 1.0 by definition, ensuring the sequence begins with a boundary"
- The boundary probability measures dissimilarity: a value near 1.0 means the current token is very different from the previous one (boundary likely); a value near 0.0 means they are similar (no boundary).
